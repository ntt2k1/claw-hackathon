3 Hướng Kiến Trúc

Chọn cách xây dựng hệ thống — từ đơn giản đến phức tạp

A

Rule-based scoring + LLM gợi ý

Frontend React tự tính điểm (JS thuần), chỉ gọi LLM 1 lần ở màn 5 để sinh địa điểm + lịch trình. Backend FastAPI tối giản.

Pros

  • Dev nhanh, ít lỗi
  • Chi phí token thấp
  • Dễ debug scoring

Cons

  • LangChain chỉ dùng như wrapper đơn giản
  • Ít "agent" thực sự
B

LangChain Agent với Tools ⭐ Recommended

Backend LangChain Agent có 3 tools: score_personality(answers) → search_locations(vibe, location, trip_type) → build_itinerary(places, hours/days). Frontend gọi 1 endpoint duy nhất, agent tự điều phối.

Pros

  • Đúng tinh thần "AI Agent"
  • Flexible — agent tự quyết cần gọi tool nào
  • Dễ mở rộng thêm tool sau
  • Phù hợp demo hackathon

Cons

  • Phức tạp hơn A một chút
  • Tốn token hơn
C

LangChain Agent + Streaming + Memory

Giống B nhưng thêm streaming response (typewriter effect ở màn 5) và ConversationMemory để user có thể chat tiếp sau khi xem kết quả ("đổi sang chỗ gần hơn", "thêm option ăn chay").

Pros

  • UX đẹp nhất (typewriter)
  • Interactive sau kết quả

Cons

  • Dev phức tạp hơn đáng kể
  • Rủi ro cao hơn cho hackathon
Gợi ý B — đủ "agent" để thuyết phục giám khảo, vừa sức hoàn thành trong hackathon, dễ demo. Nếu còn thời gian có thể bổ sung streaming từ B lên C.
